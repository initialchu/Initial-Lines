# 初笺 V0.1 设计说明

**日期：** 2026-05-23
**状态：** 设计完成，待实施

---

## 1. 概述

初笺 V0.1 聚焦核心编辑体验：基于 CodeMirror 6 的 markdown 编辑器，支持编辑/预览切换和基本文件操作。

## 2. 技术选型

| 层 | 选择 | 理由 |
|---|------|------|
| 编辑器核心 | CodeMirror 6 | 模块化、markdown 原生支持、Tauri 生态成熟 |
| 预览渲染 | markdown-it + highlight.js | 中文支持好、插件丰富、Vue 生态常用 |
| 前端框架 | Vue 3 + TypeScript + Vite | 既定技术栈 |
| 后端 | Rust + Tauri v2 | 既定技术栈 |

## 3. 组件架构

```
App.vue
├── TitleBar.vue          # 标题栏：当前文件名 + 窗口控制按钮
├── Toolbar.vue           # 工具栏：新建 | 打开 | 保存 | 另存为 | 编辑/预览切换按钮
└── EditorArea.vue        # 内容区，根据状态展示一个
    ├── CodeEditor.vue    # CodeMirror 6 编辑态
    └── MarkdownPreview.vue  # markdown-it 渲染预览态
```

## 4. 数据流

- `App.vue` 持有：`currentFilePath: string | null`、`currentContent: string`
- `Toolbar.vue` 通过 emit 通知父组件文件操作（新建、打开、保存、另存为）
- `EditorArea.vue` 接收 `content` prop，通过 emit 回传 `update:content` 变更
- 文件 I/O 通过 Tauri `invoke()` 调用 Rust 命令完成
- 编辑/预览切换是纯 UI 状态（`showPreview: boolean`），不改变内容

## 5. Rust 端接口

```rust
// 计划新增的 Tauri commands
#[tauri::command] fn open_file() -> Option<(String, String)>;       // 返回 (路径, 内容)
#[tauri::command] fn save_file(path: &str, content: &str);          // 保存到指定路径
#[tauri::command] fn save_file_as(content: &str) -> Option<String>;  // 另存为，返回新路径
```

## 6. 文件操作流程

- **新建：** 清空内容 + 路径置空，提示用户保存时选择路径
- **打开：** 调用 Tauri 原生文件对话框打开 .md/.txt 文件
- **保存：** 如果有当前路径则直接写入，否则触发"另存为"
- **另存为：** 弹出原生保存对话框，用户选择路径和文件类型
